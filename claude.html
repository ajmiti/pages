For everyone

Eight easy habits

No technical setup required. These work in Claude chat and Cowork and pay off immediately.

add_comment

Start a fresh chat for a new topic

Every reply re-reads the whole thread. A chat that has drifted across five topics is expensive on every turn.

Do this: Finished one task? Open a new chat for the next one.
center_focus_strong

Front-load a clear, specific prompt

One detailed request reaches the answer faster than ten vague follow-ups, and each follow-up costs tokens.

Do this: Say the goal, format, and any constraints up front.
content_cut

Paste only what's needed

Dropping in a 50-page document to ask about one table makes Claude read all 50 pages every time.

Do this: Share the relevant section, not the whole file.
tune

Match the model to the job

A lighter, cheaper model handles summaries, formatting, and quick questions perfectly. Save the heavyweight model for hard reasoning.

Do this: Default to a faster model; switch up only when quality needs it. New to the models? See the model cheat sheet.
bookmark

Reuse prompts, don't rebuild them

Re-explaining the same context each time burns tokens. Save prompts that work well.

Do this: Keep a notes file of your best prompts and paste them in.
folder_special

Use Projects to set context once

A Project is a dedicated space where you add your reference docs and instructions a single time. Every chat inside it can draw on them, so you don't re-paste the same background on each new conversation.

Do this: For recurring work, create a Project and load the key docs and guidelines there.
refresh

Refine instead of "try again"

Hitting regenerate repeatedly re-runs the full cost each time without telling Claude what to fix.

Do this: Say what was wrong, like "shorter, more formal," in one edit.
short_text

Ask for the length you need

Output tokens cost more than input. If you want three bullets, say so instead of getting three pages.

Do this: Add "keep it brief" or "just the final answer" when that's all you need.
For power users

Level up

For heavy users, Claude Code, and anyone building repeatable workflows. More control, more savings.

layers

Default down, escalate on demand

Pick the smallest model that can do the job and only step up when output quality actually requires it, not by habit.

Do this: Use the model cheat sheet to route work to the right tier.
cached

Use prompt caching for repeated context

When many calls share the same large system prompt or reference doc, caching the stable part can sharply cut input-token cost.

Do this: Put the unchanging context first so it can be reused across calls.
cleaning_services

Clear and compact in Claude Code

Long sessions accumulate context that's re-sent every turn. Resetting keeps each request lean.

Do this: Run /clear between unrelated tasks and /compact on long sessions.
checklist

Batch related questions

Five separate chats each re-establish context. One prompt with five questions shares it.

Do this: Group related asks into a single, structured prompt.
find_in_page

Scope what gets loaded

Pointing at a whole repo or folder loads far more than the task needs. Be precise about files.

Do this: Reference specific files; use subagents for big searches so dumps don't bloat context.
functions

Use the right tool for the job

Crunching a large dataset row-by-row through the model is slow and costly. A script or formula is exact and nearly free.

Do this: Let Claude write the code, then run it on the data.
Cheat sheet

Which model for which task

The tiers all understand plain language and follow instructions. They mainly differ in how much horsepower (and cost) they bring. Picking a heavier model than a task needs is the most common way to overspend, so reach for the lightest tier that does the job well.

ModelWhat it's likeBest for
HaikuFastest and lowest costSummaries, formatting, extraction, quick Q&A, classification
SonnetBalanced quality and cost, the sensible defaultMost drafting, everyday coding, analysis, research
OpusMost capable and most expensiveComplex reasoning, hard debugging, architecture, nuanced writing
savings

The one rule to remember

Shorter context, sharper prompts, smaller model. Reach for more only when the task truly needs it.

trending_up Keep watch

AI tools are a moving target

These tools don't behave like traditional software. Anthropic and other providers can ship changes, like dynamic workflows or subagents that spawn more work, that shift how many tokens a task burns, often overnight.

There may be no visible change to the interface and no notification. The product can look identical from one day to the next yet cost 5–20× more to run the same task.

Action item

Check your spend daily. A quick glance at usage catches a sudden jump early, not at the end of the billing cycle.
